# widget_bank
> ⚠️ **Важно:** Все сгенерированные номера карт — **тестовые**, не являются валидными платёжными инструментами


## Цель проекта: 
освоить элементарные команды для работы с GIT


## Описание:

Главный файл для запуска: *src/widget.py*
В файле собран запуск всех имеющихся модулей, результат выводится в консоли


## Инструкция для запуска из консоли

Скопируйте и в консоли выполните команду : 

```bash
python3.12 src/widget.py
```

## Тесты 
Добавлены unit-тесты для всех функций проекта.
Тестирование выполняется с помощью фреймворка <span style="font-size: 120%; color: green; font-family: monospace;">pytest</span>
Дополнительно установлена библиотека **pytest-cov** для оценки процента тестирования

Запуск тестов выполняется командой:
```bash
pytest --cov
```

## Входные тестовые данные содержаться в файле *data/transactions.json*
Файл подключается как при работе приложения, так и при тестирования. Иммитирует данные о текущих транзакциях
- **Загрузка транзакций** из `data/transactions.json` с безопасной обработкой отсутствия файла и ошибок формата JSON.
- **Фильтрация по валюте** (`filter_by_currency`) через генератор — экономит память при работе с большими наборами данных.
- **Извлечение описаний операций** (`transaction_descriptions`) — ленивая выдача строк без создания промежуточных списков.
- **Генерация последовательных номеров карт** (`card_number_generator`) в формате `XXXX XXXX XXXX XXXX` для использования в тестах (например, для `get_mask_card_number`).
- **Устойчивость к битым данным**: пропуски транзакций с отсутствующими полями вместо падения скрипта.
